    Tri-Dimensional Chess — 핵심 룰 요약

    보드 구조

    • 메인 보드 3장 (각 4×4): W (백 측, 가장 아래) → N (중립, 중간) → B (흑 측, 가장 위). 각 다음 보드가 이전 보드의 rank 3 위에서 시작 — 2 rank 씩 겹치는 staircase.
    • 어택 보드 4장 (각 2×2): QL1/KL1 은 W 의 좌·우 모서리에, QL3/KL3 는 B 의 모서리에 부착. 어택 보드의 한 칸과 부모 메인 보드의 한 칸이 같은 (x, z) 좌표 공유.
    • 총 64 칸 — 표준 체스와 같지만 7 레벨에 분산.

    이동 규칙 (요약)

    • Pawn: 전진 1 (첫 이동 시 2), 대각 캡처. 보드 간 진출 시 abs rank 가 같은 다른 레벨로도 이동 가능 (플레이어 선택).
    • Knight: 3D L-shape — 8 종 (±2, ±1)/(±1, ±2) 점프. 경로의 장애물 무시.
    • Bishop: 대각 슬라이딩 (4 방향). 보드 간 이동 가능.
    • Rook: 직선 슬라이딩 (4 방향). 보드 간 이동 가능.
    • Queen: Rook + Bishop.
    • King: 인접 1 칸 + 같은 (x,z) 좌표의 다른 레벨로 수직 이동.

    3D 핵심 — Highest Path

    한 abs 좌표 cell 에 여러 레벨의 칸이 동시에 존재할 수 있음 (보드 겹침). 플레이어가 도착 레벨을 선택, 그 사이 경로는 룰이 자동 결정. 본 프로젝트는 multi-candidate 단순화 적용.

    Attack Board 이동 (Pin 모델)

    • 턴에 piece 대신 자기 AB 자체를 이동 가능 (탑승: 비었거나 자기 폰 1개).
    • 핀(Pin) 기반: 각 메인 보드의 4 모서리 × 위/아래 = file 당 12 pins. AB 는 핀에 부착.
    • 이동 거리: 현재 핀에서 rank 거리 ≤ 2 인 다른 핀으로. 핀 ID 예: W-S-up, N-S-dn.
    • 색깔 제약: 회전 4개 중 2개만 합법 (180° flip).
    • 반전·소유권 이전: 후속 sprint 예정.

    특수 룰 (캐슬링·앙파상·프로모션)

    • 3D 캐슬링: King-side 자리 교체, Queen-side teleport.
    • 앙파상: 표준 체스 그대로, "use it or lose it".
    • 프로모션: 백은 rank 8/9, 흑은 rank 1/0 도달 시.
    본 구현의 한계 (MVP): Attack Board 이동 미구현 (M4), 캐슬링 / 앙파상 / 프로모션 미구현 (M5), 자기 체크 차단 미구현 (Sprint 3.5). Highest-path 도 정확한 path 검증 대신 multi-candidate 단순화 (ADR-0010).
